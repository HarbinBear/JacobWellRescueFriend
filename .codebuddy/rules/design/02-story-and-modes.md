# 设计文档分卷 02：剧情结构、关键事件与模式

## 本卷用途

本卷主要回答下面这些问题：

- 主线是按什么章节推进的
- 每一关的情绪目标是什么
- 哪些剧情事件是核心装置
- 主线、竞技场、菜单这几种模式各自承担什么角色
- 结局与失败在设计上分别意味着什么

如果任务与剧情、分支、关卡节奏、模式定位有关，应优先阅读本卷。

---

## 一、章节结构与剧情阶段

### 1.1 `stage` 是主线剧情的核心驱动轴

项目中主线剧情以 `state.story.stage` 为主导。模型接手时应优先把 `stage` 视作：

- 当前故事位于哪一章
- 哪些事件应该允许触发
- 哪些角色该出现
- 哪些 UI 应显示
- 哪些结局分支已开放

### 1.2 各章节语义

| 章节 | `stage` 范围 | 设计语义 |
|------|--------------|----------|
| 第一关 | 1~2 | 初次下潜，建立世界、建立目标、建立“进不去”的挫败感 |
| 第二关 | 3~6 | 二次下潜，真正进入危险区域，体验“被卡住濒死” |
| 第三关 | 7~8 | 独自深入，灯光异常、恐怖演出、放弃与坚持的道德压力 |
| 第四关 | 9 | 更深层的寻找与真相区域，当前仍在开发中 |

### 1.3 第一关：建立目标与阻隔

第一关设计任务：

- 告诉玩家为什么要下潜。
- 把熊子作为明确目标对象立起来。
- 让玩家第一次看到关键狭缝区域。
- 通过坍塌或阻挡机制明确告诉玩家：**你现在还进不去**。

关键作用不是完成救援，而是建立“必须再来一次”的动机。

### 1.4 第二关：把危险变成亲身体验

第二关的设计重点是将第一关“看到危险”，升级为“自己成为危险的受害者”：

- 玩家更深入。
- 进入狭窄通道。
- 被卡住。
- 氧气耗尽。
- 视野收窄与红屏加深。
- 最终由潘子救出。

这是主线里非常关键的一段，因为它不只是剧情文本，而是对洞穴潜水风险的**玩法式说明**。

### 1.5 第三关：主观恐怖与道德选择

第三关的设计气质和前两关不同，重点不是纯粹物理危险，而是：

- 独自行动带来的孤立感。
- 手电损坏造成的可靠信息源崩溃。
- 鱼眼闪现的惊吓演出。
- 灰色氧气罐与异常视觉事件。
- “长按放弃救援”的心理压迫。

第三关把“救援”从空间问题提升为**意志问题**。

### 1.6 第四关：深层目标区

第四关当前仍在开发中，但从现有地图与地标命名可以看出，它定位为：

- 更深层区域探索
- 接近最终真相或最终结果
- 真正意义上的终局区域

后续扩展时应尽量保持第三关之后的情绪承接，而不是突然转成普通探索关卡。

## 二、关键剧情事件设计

### 2.1 潜水服发现

潜水服是第一批强叙事地标之一，主要作用：

- 在视觉上提供“这里有人出过事”的证据。
- 为玩家建立环境叙事，而不仅是对白。
- 作为第一洞室的重要探索回报。

### 2.2 缝隙坍塌

这是第一关最关键的阻挡事件。

设计目的：

- 明确后续目标位置。
- 阻止玩家过早进入后段剧情。
- 让熊子“消失在你够不到的地方”。
- 形成心理债务：你已经看到他进去，但你没法立刻救他。

### 2.3 濒死体验

这是第二关的核心体验节点。

表现层通常包括：

- 移动受阻
- 氧气接近归零
- 屏幕红色叠加
- 视野收缩
- 压力持续上升
- NPC 救援介入

它是整个项目最能体现“潜水恐惧”的玩法桥段之一。

### 2.4 手电筒损坏

手电是玩家在黑暗中最重要的控制权来源，因此它损坏的效果不只是减弱光照，而是削弱玩家的心理安全感。

手电损坏后，系统层面会影响：

- 可见性
- 光照稳定性
- 对凶猛鱼的威慑效果
- 后续恐怖演出的可信度

### 2.5 鱼眼闪现

鱼眼演出属于典型的“短时高强度惊吓点”，设计上应理解为：

- 不是普通敌人出场
- 不是常规战斗触发
- 而是环境异常与深水恐惧的具象化表达

它要服务的是情绪峰值，而不是伤害数值。

### 2.6 放弃救援按钮

“长按放弃救援”是一个非常重要的设计装置，它的作用不是给玩家一个方便退出键，而是把“撤退”显性化、道德化。

其设计价值包括：

- 让玩家明确意识到自己在做选择。
- 让“放弃”变成需要主动承担的行为。
- 通过长按而非轻触，增加心理重量。
- 让结局分支具有参与感，而不是被动播放。

### 2.7 熊子死亡结局

当前至少存在两种可达成方式：

1. 长按放弃救援。
2. 第三关手电损坏后选择上岸。

在设计上，这两种本质都属于“未完成救援”的失败型叙事分支，只是心理含义不同：

- 主动放弃：更直白。
- 转身离开：更克制，但同样残酷。

## 三、游戏模式设计

### 3.1 主线剧情模式

主线模式由 `state.screen = 'play'` 表示。

它包含以下核心机制：

- 洞穴地图移动
- 氧气消耗与生存压力
- 氮气或深潜风险管理
- 光照与视野限制
- 剧情触发
- NPC 协作或跟随
- 引导绳部署
- 特定敌对或惊吓事件

主线模式最重要的不是“全系统都一直启用”，而是**根据章节阶段有选择地启用事件与规则**。

### 3.2 食人鱼纯享版竞技场

竞技场模式由 `state.screen = 'fishArena'` 表示，是一个从主线体系中分离出来的动作生存玩法。

特点：

- 无限氧气
- 无主线剧情推进
- 无复杂 NPC 同行逻辑
- 小而封闭的战斗地图
- 以波次敌人与清场节奏为中心

竞技场的存在意义：

- 复用凶猛鱼与攻击系统
- 提供一个与主线完全不同的即时反馈模式
- 增加项目可玩性与重复游玩价值
- 允许在不破坏主线节奏的前提下测试敌人与战斗手感

### 3.3 迷宫引导绳模式（多次下潜闭环）

迷宫引导绳模式由 `state.screen = 'mazeRescue'` 表示，已从一次性迷宫升级为**多次下潜闭环**模式。

#### 核心闭环

```
岸上准备 → 下潜探路 → 主动撤离或氧气耗尽 → 返岸结算 → 岸上整理 → 再次下潜 → ... → 正式救援 → 带人上浮
```

#### 阶段说明

- **岸上阶段（shore）**：查看探索记录面板，点击认知地图图标可全屏查看手绘地图，点击水面洞口直接下潜
- **入水动效（diving_in）**：点击洞口后播放约1.5秒的入水过渡动画（岸上涟漪扩散 → 蓝色水面覆盖 → 水下深蓝气泡），然后进入正式游戏
- **水下游戏（play）**：探索未知区域、铺设绳索、发现NPC位置；未带人时可随时长按撤离按钮安全上浮（不区分探路和救援，只要没带人就能撤）；发现NPC后可靠近长按绑绳救援
- **返岸结算（debrief）**：展示本次下潜成果（用时、深度、新探索、新绳索），然后回到岸上

#### 跨下潜保留的数据

- 同一张迷宫地图
- 已探索区域（累积）
- 已铺设绳索
- 是否已发现NPC位置
- 历史最深到达
- 每次下潜摘要记录

#### 关键设计约束

- 氧气耗尽不是死亡，而是被迫返回岸上（保留所有成果）
- 不区分"侦察"和"正式救援"按钮，统一通过点击洞口下潜
- 系统根据是否已发现NPC自动决定下潜类型（未发现→scout，已发现→rescue）
- 发现NPC后任何下潜都可以靠近长按绑绳救援，不需要切换模式
- 绳索跨下潜保留，是玩家给未来自己铺的基础设施

#### 地图特点

- 每局随机生成**洞穴式迷宫地图**，整体由大量小中型洞袋、弯折通道、回环连接和深层死路组成
- 正确路线藏在全图级别的分叉、回折、回环和假捷径里
- 空间节奏在压抑窄路和受控大洞室之间交替
- 结果质量必须可验证，坏图自动重生成

#### 岸上界面

- 绿草地、阳光、树林的温馨场景
- 水面洞口作为下潜入口（带呼吸光晕和"点击下潜"提示），洞口本身就是按钮
- 信息卡片展示探索记录（下潜次数、绳索、最深到达、被困者状态、上次下潜摘要）
- 认知地图图标（🗺️）位于卡片右上角，点击可打开全屏认知地图查看页面
- 全屏认知地图：手绘风格纸张底色（带折痕和斑点纹理）、不规则笔迹绘制的已探索区域（水彩蓝色水域+铅笔棕色岩壁）、手绘虚线绳索路径、手绘标记（出口三角+被困者X标记）、图例，点击任意位置关闭

迷宫模式的存在意义：

- 验证多次下潜闭环的核心体验
- 验证引导绳在复杂地形中的跨下潜价值
- 验证"认知积累"作为核心玩法的可行性
- 为后续主线中更复杂的绳索与探索玩法提供设计依据
### 3.4 菜单系统

菜单以 `state.screen = 'menu'` 为总入口，并通过 `state.menuScreen` 区分子页面。

常见子页面：

- `main`：主菜单
- `chapter`：章节选择

菜单除了展示按钮外，还承担：

- 切换游戏模式
- 控制哪些入口可用
- 按项目阶段开放章节
- 在特殊模式下禁用主线入口

### 3.4 特殊模式开关

`fishArenaMode = true` 时，主线入口置灰，只开放竞技场。

这说明该项目存在一种“演示或限制模式”思路：

- 可以把版本裁成纯竞技场体验
- 可以临时禁用未准备好的主线内容
- 可以作为测试或发布策略的一部分

## 四、结局与失败设计

### 4.1 失败不等于游戏结束语义

在这个项目中，“失败”有两种含义：

1. **系统失败**：例如被凶猛鱼击杀、氧气耗尽。
2. **叙事失败**：例如放弃救援、回到水面导致熊子死亡。

两者都可能结束当前流程，但情绪含义不同。

### 4.2 结局的设计价值

项目中的结局不是单纯奖励，而是为了强化主题：

- 救援是否值得继续
- 你能承受多深的风险
- 你是否真的敢面对洞穴深处
- 有些下潜即使完成，也未必是真正意义上的胜利

### 4.3 第三关分支的重要性

第三关是当前最明确的选择分支关卡，因此后续新增分支时，应尽量遵循它的设计原则：

- 决策应嵌入情绪高压场景
- 决策应有动作成本
- 决策结果应具有叙事后果

---

## 五、本卷最重要的结论

- **主线是多次下潜的章节结构，不是一次性长流程。**
- **第二关核心是濒死体验，第三关核心是主观恐怖与放弃的重量。**
- **放弃救援不是普通退出键，而是被设计成有心理成本的叙事选择。**
- **竞技场是独立玩法模式，但不能主导主线叙事方向。**
