# 设计文档分卷 08：唐老师叙事蓝本（未来主线目标形态）

## 本卷用途

本卷记录一份**更完整、更成熟的主线叙事蓝本**，暂称《唐老师的救援》。

后续接手项目的 AI 必须明白以下两件事：

1. 这份叙事是**项目主线未来的目标形态**，不是当前已落地的内容。
2. 当前代码里已经实现的"熊子 / 潘子 / 四关结构 / 雅各布井"是这份叙事的**早期雏形**，两者不冲突，而是同一个故事在两个完成度阶段的切片。

如果未来要对主线做方向性重构，应该把本卷视为**目标锚点**，而不是推倒重来的新故事。

## 二、故事梗概（完整版）

### 2.1 前期：背景设定与新手引导

#### 2.1.1 主线

故事主人公是一位年长男性（暂称**唐老师**），身份是林区护林员。

- 这片林区是著名的洞穴潜水地点，潜水爱好者慕名而来，水下事故频发
- 他日常接到潜水救援电话就下水救人
- 他记性不好，对自己以前的事情说不太上来

玩法从简单到复杂逐层铺垫：

- **最简单**：有已探明地图、有预布置引导绳的洞穴，完成遗体打捞
- **加泥沙遮挡**：视线变差，玩家开始体会环境压迫
- **加绳索引导玩法**：必须自己布绳、看绳回家

#### 2.1.2 女孩线与梦境线

这是独立于主线救援的第二条线。

**女孩线**：
- 晚上回住所，总有一个小女孩来找他
- 女孩起初说丢了一个玩具"会跳舞的**布拉拉**"，坚称一定在男主家里
- 女孩叮嘱男主："如果找到她，不要让她一个人呆在黑的地方。"
- 男主当成小孩过家家，不当真

**梦境线**：
- 男主睡觉总梦到：自己在黑暗洞穴里没气快死，身边的同伴看了自己一眼就走了
- 梦里看不清同伴的脸
- 每次完成任务后再做这个梦，梦境每次会往前推一点点

### 2.2 中期：玩法拓展，埋感情线

#### 2.2.1 主线

- 之前都是有预布置绳 + 有地图的已探明洞穴；**从这里开始**进入无引导绳、无地图的未探明洞穴
- 加入凶猛洞穴食人鱼：**怕光**，玩家可以用手电照着它前进
- 升级：可以用潜水氧气瓶近距离突袭，砸晕食人鱼使其短时失能
- 再升级：被砸晕的食人鱼可以用引导绳绑住，永久无害化
- 每次救援都基于**真实洞穴潜水救援案例**改编，完成后男主写**救援报告**

#### 2.2.2 女孩线与梦境线

女孩线和梦境线主要承担**缓解主线救援疲劳 + 推进叙事**的双重作用。

- 万圣节女孩来 trick or treat，自己也 cosplay 成潜水员来逗男主开心
- 布拉拉仍然没找到，女孩反复叮嘱"一定要照看好她"
- 男主仍然烦这个莫名其妙的陌生女孩，但有意识地回避让她看到血腥场面（比如他处理过的尸体）
- **细节暗示**：
  - 女孩居然懂潜水手势，还知道"不要踢水底泥沙"——这个年纪本不该懂
  - 女孩问："你家的音乐停了好久。"
- 某天男主一次救援失败，还和遇难者家属发生争执，情绪极差时碰上女孩做了些熊孩子的事，对女孩爆发
- 女孩哭着走了，之后再也没来过

### 2.3 后期：现实崩塌

#### 2.3.1 主线的玩法深化方向

- **通过狭缝**：必须卸装备通过
- **假通路**：看似能走，其实进死胡同
- **水流突变**：逆流耗氧加快
- **绳索缠绕**：慌乱越挣扎越糟
- **天花板落石**：制造动态路线变化
- **氧气系统故障**：临时切换气源 / 触发幻觉
- **失温 / 疲劳**：操作延迟、动作不稳
- **尸体 / 残骸卡点**：心理压迫
- **灯光故障**：备用光源管理
- **幻觉**：水下超现实恐怖战斗景象
- **盲鱼群**：不作为战斗对象的生物背景
- **与救援对象互动**：共享气瓶、安抚、指挥、限制其移动
- **任务前自定义装备**

#### 2.3.2 女孩线与梦境线的崩塌

- 男主向同事打听小女孩，发现**根本没人见过她**
- 同事说"这荒山野岭的，哪来的小女孩？不就我们几个大老爷们护林员吗？"
- 男主也再没见过她
- 梦境最终推进到**最关键一幕**：那个抛下同伴见死不救的人，是年少的男主自己
- 主线最后一次潜水地点是**雅各布井**，和梦境里的洞穴一点点对上

### 2.4 结局：回忆杀、完成救赎

男主在雅各布井彻底想起当年。

#### 2.4.1 真相

- 年少时他有一个一起长大、一起潜水的女孩
- 那年他们一起被困在这个洞底
- 男主当时想"我先上去带人下来救你"，结果上去后引发塌方或被泥沙阻挡，**找不到回去的路**
- 最后他自己在缺氧绝望中晕厥获救
- 梦里那个"看了一眼就走的人"，**是他无法原谅的自己**
- 他因极端创伤触发**解离性失忆**，忘了女孩，忘了半生
- 只剩一个执念："我必须去做潜水救援"——每次"救援报告"其实都是**他给自己的赎罪案卷**

#### 2.4.2 收尾

- 最后一次救援重演当年场景：他和被救者再次困在"假烟囱"里
- 他把唯一的生机让给被救者，自己留下
- 临终前在水底看到小女孩一直在找的"会跳舞的布拉拉"——其实是一个**八音盒**，是年少女友的旧玩具
- 男主抱着玩具安详赴死
- 手里生锈的玩具变成崭新的，开始转动发出音乐（建议类似《To the Moon》里的 *For River*）
- 动态衔接到回忆杀：年少的自己和女友在梦境中重逢
- **洞穴潜水线 / 女孩线 / 梦境线三条线合一**

#### 2.4.3 玩家可感知的真相节奏

玩家应该能**比较早**猜到"女孩 = 过去的投影"，但直到最后才知道：

- 女孩**不是**女友本人，而是**女友童年记忆的投影**
- 布拉拉不是普通玩具，是当年事故前两人共享的重要物件
- 男主不是"见死不救"，而是"试图求救却造成不可逆后果"
- 他遗忘的不是"一次事故"，而是**求救后回来找不到她、反复下潜失败、最终昏迷获救**这一整段过程
- 救援报告不是工作记录，而是他在反复重写那一份失败案卷

### 2.5 结局分支

| 结局 | 触发条件 | 语义 |
|---|---|---|
| 普通结局 | 救出他人，自己留在水下 | 与记忆和解，但生命留在雅各布井 |
| **隐藏完美结局** | 前期救援报告 / 遗物收集 / 梦境线 / 装备维护累积足够条件 | 男主也能生还，但必须面对完整现实 |

## 三、设计母题（必须守住的核心）

这些母题是本叙事的"DNA"，后续任何落地实现都不能破坏：

1. **救援是自我赎罪的外化**：不是工作，是强迫性重复创伤
2. **记忆不可靠**：男主的所有认知都要允许被后期推翻
3. **女孩必须鲜活**：女孩不能沦为催泪工具，她要通过"房间物件 / 讲过的话 / 知道潜水手势 / 知道不要踢泥沙"等生活化细节立起来
4. **三线合一**：洞穴线 / 女孩线 / 梦境线必须在结局那一刻同时收束，任何一条单独存在都应成立
5. **音乐 = 时间锚**：八音盒 For River 式主题曲是贯穿全程的情绪载体，必须前期就用合理理由植入
6. **玩法成长曲线**：探索定位 → 风险控制 → 救援搬运 → 复杂导航 → 高压决策 → 记忆重演（**不是**"探索 → 打鱼 → 打更强的鱼"）

## 五、玩法与叙事结合的重点切片：玩家感知被救者

这一节是目前最有机会**先行落地**的叙事玩法结合点，单独拎出来讲清楚。

### 5.1 现有能力回顾

当前迷宫模式中玩家对被救者（NPC）的感知是一个单点触发：

- 距离 NPC 约 `npcRescueRange * 2 ≈ 160px` 时一次性触发"发现被困者！"
- NPC 自己在 `npcRescueRange * 3 ≈ 240px` 内有呼救气泡 / 挥手 / 闪光圈
- 除此之外无任何渐进感知层

**问题**：100×100 的迷宫里，从"不知道他在哪"到"终于到了他身边"之间**缺乏一段叙事化的渐进感知**。这段体验真空，在纯玩法角度是"引导不足"，但在本卷叙事角度是一个**极其宝贵的情绪空间**。

### 5.2 叙事结合原则

**不要把这段感知做成 GPS/雷达/方向箭头**。那是动作游戏的解法。

对本卷叙事来说，"玩家感知被救者"这件事，天然应该**承载记忆 / 梦境 / 赎罪**这三条母题：

| 感知手段 | 表层读感 | 叙事暗示 |
|---|---|---|
| 远处断续的**敲击声** | 洞穴物理 | 梦境里他听过一样的声音 |
| 中距离的**耳鸣 / 心跳** | 潜水生理 | 男主身体在认识这个地方 |
| 近距离的**尘埃朝某方向漂** | 水流扰动 | 仿佛洞穴在"指引"他 |
| 极近的**八音盒残响** | 超现实 | 真相的前兆 |

**关键**：感知层级越深，越不像"物理传感"，越像"他自己在想起什么"。这是本卷叙事独有的感知语法——**洞穴本身就是男主的潜意识**。

### 5.3 分层设计建议（与叙事绑定的版本）

| 层级 | 距离 | 玩法表现 | 叙事暗示 |
|---|---|---|---|
| **L1 远方** | 1500~2500px | 每 15~25 秒一声闷响 SFX，不定位 | "又是这个声音。梦里……好像也是？" |
| **L2 规律** | 800~1500px | 敲击变规律，约 6 秒一次 | 男主下意识加快了呼吸（氧气消耗轻微加速，不告诉玩家） |
| **L3 心跳警觉** | 400~800px | 屏幕四周暗蓝心跳光晕；敲击+模糊呼喊 | 男主的手颤了一下（手电光圈微微晃动） |
| **L4 洞穴低语** | 180~400px | 悬浮尘埃加倍并**朝 NPC 方向缓漂**；手电光锥末端偶尔泛绿闪烁 | 第一次听到八音盒的一两个音符（极轻，一闪而过） |
| **L5 发现** | <180px | 走现有"发现被困者！"触发 | 和梦里完全重合 |

每一层的叙事暗示都**不要用文字弹窗说明**，只通过声音 / 画面 / 身体反应传达。玩家也许要三四次下潜后才会意识到"那段音乐好像在哪里听过"——这正是本卷想要的节奏。

### 5.4 八音盒作为"终极导航"

到了**最后一次救援（雅各布井章）**，这套感知层应该**发生质变**：

- L1~L3 依然是物理感知
- L4 层开始，八音盒音乐会**越来越清晰**，从一两个音符变成完整旋律
- 在某一刻玩家会意识到：**引导我走向被救者的不是物理声音，是这首他压根想不起来的音乐**
- 这时候玩家回忆起前面每次下潜 L4 层听到的那段断续音符——**原来一直是她**

这套设计的精妙之处在于：**玩法层从第一关到结局完全一致（都是距离分层提示），但叙事层的含义在最后一关彻底翻转**。前面玩家以为的"洞穴回声"，全是女友八音盒的前奏。

### 5.5 增强完美结局的条件挂接

这套感知系统天然可以挂"完美结局条件计数"：

- 玩家是否在 L1/L2 层就开始**主动朝声音方向移动**（而不是随便走）→ 说明他在"听"
- 玩家在 L4 层的停留时间 → 说明他对环境异常敏感
- 玩家在最后一关前是否听全过 8 个音符 → 记忆完整度

这些数据都可以**静默累积**，不告诉玩家，但在最后判定完美结局时一并使用。

### 5.6 先行落地建议

如果要现在就开始做这个能力，建议按下面顺序：

1. **先做纯物理层**（L1~L5 声音 + 光晕 + 尘埃漂向）——不带任何叙事元素
2. **再接入音乐层**——L4 的"泛绿闪烁"替换为一两个八音盒音符（需要 AudioManager 支持一段极短的音乐片段）
3. **最后接入计数器**——为未来完美结局埋点

第 1 步是纯玩法需求，和当前四关主线无缝兼容；第 2、3 步需要等本卷正式立项时再推。

## 六、落地路线建议

### 6.1 短期（当前阶段）
