# 设计文档分卷 03：世界结构与核心玩法系统

## 本卷用途

本卷主要回答下面这些问题：

- 地图为什么是现在这种结构
- 区域和地标在设计上承担什么作用
- 玩家为什么会持续感到压力
- 绳索、手电、氧气、凶猛鱼分别服务什么体验
- NPC 在主线中为什么重要

如果任务与地图、地标、生存压力、敌人、绳索有关，应优先阅读本卷。

---

## 一、地图系统设计

### 1.1 主线地图基础规格

主线地图是固定洞穴结构，而非程序化随机图。

参数：

- 地图尺寸：80 列 × 150 行
- 单格大小：40 × 40 像素
- 总尺寸：3200 × 6000 像素

这种纵向长地图非常适合表达“向下深入”的心理感受。

### 1.2 地图数据来源

地图并非纯手写数组，而是通过素材图片处理生成：

- 原始视觉资源：`texture/Well.png`
- 处理脚本：`scripts/processMap.py`
- 产出数据：`src/world/mapData.ts`

这意味着地图编辑工作流更偏向：

- 先画轮廓或区域图
- 再由脚本转成游戏内可用数据

优点：

- 对关卡形状编辑直观
- 便于迭代整体洞穴结构
- 适合项目这种地形主导型玩法

### 1.3 地图格子语义

项目中至少有以下几种格子或碰撞语义：

- `0`：水域，可通行
- `1`：生成过程中的岩石标记，后续会转换
- `wall 对象`：边缘可视岩石，用于渲染圆润墙面与碰撞
- `2`：内部实体岩石，主要服务碰撞，通常不作为圆形边缘渲染

这个设计体现出：

- **渲染墙体**和**碰撞墙体**不是完全同一层抽象。
- 地图最终不是纯数字数组，而是混合数据结构。

### 1.4 区域划分 `state.zones`

区域划分不是装饰性命名，而是用来帮助理解：

- 当前玩家位于哪一段叙事空间
- 哪些剧情应在附近发生
- 哪些路径是主路、支路、死路

现有区域包括：

| 区域名 | 设计语义 |
|--------|----------|
| `entrance` | 水面入口与安全感残留区域 |
| `chamber1` | 早期探索区，较宽，作为进入深处前的缓冲 |
| `suit_tunnel` | 导向潜水服地标的狭窄通道 |
| `chamber2` | 更深入、更偏移的中段洞室 |
| `junction` | 路径决策区，连接多个方向 |
| `dead_end` | 用于误导、恐惧与 NPC 行动演出的假终点 |
| `chamber3` | 深层剧情区，承接第三关恐怖事件 |
| `story_tunnel` | 极窄、专门为剧情压迫感服务的通道 |
| `chamber4` | 终局潜在区域，当前开发中 |

### 1.5 关键地标 `state.landmarks`

地标是剧情与地图绑定的核心纽带。

| 地标 | 用途 |
|------|------|
| `entrance` | 入口出生点，通常自动扫描确定 |
| `suit` | 废弃潜水服位置，触发环境叙事 |
| `tunnelEntry` | 缝隙入口，第一关阻挡、第二关进入 |
| `tunnelEnd` | 玩家被卡住位置，濒死剧情核心点 |
| `deadEndDeep` | 假烟囱深处，服务第二关 NPC 行为 |
| `chamber12Junction` | 第一二洞室连接点，第三关灯损坏触发 |
| `chamber23Junction` | 二三洞室连接点，恐怖事件关键区域 |

### 1.6 地图设计意图

从区域与地标分布可以看出，主线地图并不是“大世界开放探索”，而是：

- 有一定自由度
- 但核心剧情路线上高度引导
- 关键事件强依赖具体坐标
- 空间结构服务情绪爬升

因此后续改图时要极其注意：

- 不要轻易改坏地标相对关系
- 不要让关键剧情点之间的路程节奏失真
- 不要把本应狭窄压迫的路径改成宽阔安全区

## 二、竞技场地图设计

### 2.1 竞技场基础规格

竞技场是独立正方形场地：

- 2000 × 2000 像素
- 格子大小 40 × 40
- 外围存在厚岩石墙

### 2.2 障碍物与空间目标

内部会随机生成矩形障碍物，且会避免离出生点太近。

作用：

- 提供绕行与拉扯空间
- 避免纯空场导致敌人行为单调
- 让玩家在被追逐时有可利用地形
- 让光照与近战攻击产生空间选择

### 2.3 竞技场设计语义

与主线不同，竞技场地图不需要讲故事，它要保证的是：

- 出生安全
- 敌人生成合理
- 路径可周旋
- 小地图信息清晰
- 节奏适合反复开局

因此竞技场全图默认已探索，是一个明确的“玩法场”而不是“叙事场”。

## 三、玩家核心生存系统

### 3.1 移动与朝向

玩家在俯视角下移动，朝向会影响：

- 手电照射方向
- 攻击方向
- 角色绘制方向
- 与敌人或场景的互动感受

在这种题材中，“朝向”不仅是动作参数，也是感知参数。

### 3.2 氧气系统

氧气是项目最基础的生存资源。

它承担的设计作用包括：

- 限制探索深度与停留时间
- 强迫玩家考虑回程
- 在狭窄地带制造焦虑
- 为濒死演出提供硬逻辑支撑

主线中氧气是压力来源，竞技场中则被解除，用于让战斗模式更纯粹。

### 3.3 氮气或深潜压力

虽然项目主打氧气管理，但氮气或深潜相关状态同样服务于“潜水不是简单血条”的概念。

它强调：

- 深度不是纯视觉变化，而是系统性风险变化。
- 玩家在深处承受的不仅是空间威胁，还有身体负担。

### 3.4 手电系统

手电是整个项目中最关键的感知工具之一，兼具：

- 照明
- 导航
- 氛围塑造
- 危险揭示
- 敌人威慑

它并不是简单的画面特效，而是直接影响玩法与情绪的主系统。

### 3.5 受伤、濒死与失败表现

项目中的失败表现通常会通过以下内容组合体现：

- 红屏叠加
- 屏幕震动
- 视野收缩
- 控制权下降
- 失败提示文本

这使得失败不只是一个布尔结果，而是一个逐步逼近的过程。

## 四、引导绳系统设计

### 4.1 引导绳的题材意义

洞穴潜水中的引导绳非常关键，因此它在本项目里既有真实性，也有玩法价值。

它的意义在于：

- 体现专业潜水语境
- 帮助玩家建立路线感
- 在视觉上强化“我来过这里”
- 让深处探索更有仪式感

### 4.2 交互方式

当前绳索交互不是即时点击，而是结合：

- 靠近墙体锚点
- 玩家保持相对静止
- UI 提示出现
- 长按完成起始或结束固定

这种交互方式的设计价值：

- 防止高速移动时误触
- 让“布绳”显得更像一个实际操作动作
- 让锚定行为具有明确意图感

### 4.3 锚点选择规则

绳索系统会在玩家附近寻找合适墙体，并把绳端固定在墙体表面锚点，而不是墙心。

这意味着系统关心的是：

- 视觉合理性
- 绳索与墙体的贴合感
- 路径避障后的可读性

### 4.4 绳索路径

绳索不是简单直线，而会构建一条避开障碍的路径。

这很重要，因为洞穴地图大量存在狭窄墙体结构。若直接画直线，会造成：

- 穿墙
- 视觉不可信
- 玩家误判路线

因此引导绳实际上也是一个轻量路径生成系统的表现层应用。

并且有一条很重要的体验约束：

- 玩家在结束布线前看到的预览绕线路径，完成后必须保持一致；不能出现“预览会绕岩，最终却变直线”的违和感

### 4.5 绳索状态

从设计语义上看，绳索有几种关键状态：

- **未铺设**：等待起点锚定
- **进行中**：已固定起点，尾端跟随玩家
- **收紧中**：准备固定终点时的过渡状态
- **完成**：两端固定，形成稳定引导线

### 4.6 绳索的体验作用

引导绳除了“找路”外，还有几个额外效果：

- 提升洞穴探索的专业感
- 让玩家主动建立安全感
- 让深处场景更有可追溯性
- 为画面增加空间层次

## 五、凶猛鱼与战斗系统设计

### 5.1 凶猛鱼的定位

凶猛鱼不是传统小怪，更接近一种：

- 水下危险生物
- 惊吓与压迫装置
- 带有行为模式的掠食者

在主线里，它服务氛围与局部危机；在竞技场里，它被提升为主要玩法对象。

### 5.2 行为状态机

凶猛鱼具备较细的 AI 状态：

- `roam`：自由游弋
- `detect`：发现玩家
- `stalk`：悄悄接近
- `circle`：环绕观察
- `lunge`：蓄力扑击
- `bite`：咬住玩家
- `devour`：吞食处理
- `retreat`：进食后撤离
- `fear` / `flee`：被灯光驱赶后的受惊与逃离
- `hit`：被打中或被逼退时快速撤离
- `dying`：死亡动画阶段

这套状态机说明敌人设计重点在“行为表演感”，不是简单追踪碰撞。

### 5.3 怕光机制

凶猛鱼会受到手电照射影响，但不是只要进入视野就逃跑，而是结合：

- 与玩家距离
- 在手电扇形范围内的位置
- 手电当前亮度
- 手电是否损坏闪烁

这使得手电既是照明工具，又是对敌手段。

### 5.4 扑击与吞食流程

凶猛鱼攻击有明显的节奏段落：

1. 观察或绕圈
2. 蓄力
3. 冲刺扑击
4. 咬住玩家
5. 吞食与死亡过场

这种设计让玩家有心理预警窗口，也让敌人更像捕猎者，而不是直接贴脸掉血源。

### 5.5 玩家攻击方式

玩家使用氧气瓶挥击进行近战攻击。

特点：

- 扇形判定
- 有攻击角度与范围
- 有冷却
- 有挥击表现与停留残影
- 命中后会触发明显反馈

当前敌鱼设计偏“高机动但脆弱”：常态下很难近身，但一旦被真正打中，就会进入死亡流程。

### 5.6 竞技场中的战斗定位

在竞技场里，这套系统从局部事件转化为主循环：

- 生成敌鱼
- 周旋走位
- 借障碍规避
- 抓扑击间隙反打
- 清完一波后进入下一波

因此，竞技场其实是凶猛鱼 AI 与玩家攻击系统的集中展示场。

## 六、NPC 与同伴设计

### 6.1 NPC 的主要职责

主线中的 NPC 不是纯对话对象，而是承担以下作用：

- 叙事推进
- 目标引导
- 陪伴感与对照感
- 关键救援行为
- 路线示范或误导

### 6.2 熊子

熊子是失踪者，也是驱动主线全部行动的中心角色。

他的设计功能包括：

- 作为被寻找目标提供明确任务动机
- 通过“先一步进入危险区域”制造追逐感
- 通过无法立刻救出形成持续悬念
- 通过可能死亡结局承担悲剧重心

### 6.3 潘子

潘子在第二关中的核心功能是“把危险的不可逆，暂时转回可逆”。

也就是说，他不是一般意义上的陪同 NPC，而是：

- 见证二次下潜
- 参与危险更深层区域
- 在玩家濒死时完成救援
- 让第二关形成结构上的闭环

### 6.4 NPC 的设计原则

NPC 不应喧宾夺主。

他们存在的目的不是持续占据画面中心，而是：

- 在关键时刻推动情绪变化
- 让玩家行为更有叙事对照
- 让“独自下潜”和“有人同行”的差异更明显

---

## 七、本卷最重要的结论

- **地图不是开放世界，而是服务章节节奏和情绪爬升的引导式洞穴结构。**
- **地标是剧情落地的关键纽带，改地图不能只改形状而不改地标关系。**
- **氧气、手电、狭窄地形、绳索共同构成主线压力系统。**
- **凶猛鱼在主线是危机装置，在竞技场才是主要玩法对象。**
